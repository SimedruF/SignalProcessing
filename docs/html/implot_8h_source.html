1062// Styling colors in ImPlot works similarly to styling colors in ImGui, but
1063// with one important difference. Like ImGui, all style colors are stored in an
1064// indexable array in ImPlotStyle. You can permanently modify these values through
1065// GetStyle().Colors, or temporarily modify them with Push/Pop functions below.
1066// However, by default all style colors in ImPlot default to a special color
1067// IMPLOT_AUTO_COL. The behavior of this color depends upon the style color to
1068// which it as applied:
1069//
1070// 1) For style colors associated with plot items (e.g. ImPlotCol_Line),
1071// IMPLOT_AUTO_COL tells ImPlot to color the item with the next unused
1072// color in the current colormap. Thus, every item will have a different
1073// color up to the number of colors in the colormap, at which point the
1074// colormap will roll over. For most use cases, you should not need to
1075// set these style colors to anything but IMPLOT_COL_AUTO; you are
1076// probably better off changing the current colormap. However, if you
1077// need to explicitly color a particular item you may either Push/Pop
1078// the style color around the item in question, or use the SetNextXXXStyle
1079// API below. If you permanently set one of these style colors to a specific
1080// color, or forget to call Pop, then all subsequent items will be styled
1081// with the color you set.
1082//
1083// 2) For style colors associated with plot styling (e.g. ImPlotCol_PlotBg),
1084// IMPLOT_AUTO_COL tells ImPlot to set that color from color data in your
1085// **ImGuiStyle**. The ImGuiCol_ that these style colors default to are
1086// detailed above, and in general have been mapped to produce plots visually
1087// consistent with your current ImGui style. Of course, you are free to
1088// manually set these colors to whatever you like, and further can Push/Pop
1089// them around individual plots for plot-specific styling (e.g. coloring axes).
